	ณ ห้องเรียนห้องหนึ่ง มีนักเรียนถามครูกิ่งว่า อาจารย์คะ Generalization Abstraction คืออะไรคะ ครูกิ่งก็ตอบได้ว่า เป็นความสัมพันธ์ระหว่าง
object หรือ class ในลักษณะของการสืบทอดคุณสมบัติจาก class หนึ่งไปยังอีก class หนึ่งจ้า โดยในการเขียนสัญลักษณ์แทน class 
สิ่งที่ต้องคำนึงถึงอีกสิ่งหนึ่งคือระดับการเข้าเรียกสัญลักษณ์ที่ใช้แทนการเข้าถึงนี้ว่า visibility แบ่งออกได้เป็น 3 ประเภท ได้แก่ 1. Private เขียนแทนด้วยสัญลักษณ์ - หมายถึง Attribute หรือฟังก์ชั่น
ที่ไม่สามารถมองเห็นได้จากภายนอก แต่สามารถมองเห็นได้จากภายในตัวของ class เองเท่านั้น 2. Pretect เขียนแทนด้วยสัญลักษณ์ # หมายถึง Attribute หรือฟังก์ชั่น ที่สงวนไว้
สำหรับการทำ Inheritance โดยเฉพาะ Attribute หรือฟังก์ชั่นเหล่านี้ จะเป็นของ Super class เมื่อ ทำการ Inheritance แล้ว Attribute หรือฟังก์ชั้น ที่มี visibility 
แบบ Private Attribute/ฟังก์ชั่น หรือ Protected ขึ้นอยู่กับภาษา ของ Programing ที่นำไปใช้ 3. Public เขียนแทนด้วยสัญลักษณ์ + หมายถึง Attribute หรือฟังก์ชั่น 
ที่สามารถมองเห็นได้จากภายนอก และสามารถเข้าไปเปลี่ยนค่า อ่านค่าหรือเรียกใช้งาน Attribute/ฟังก์ชั่น นั้นได้ทันที โดยอิสระจากภายนอก 
	อาจารย์คะแล้ว Associations คืออะไรคะ ครูกิ่งตอบไปอีกว่า มันเป็นความสัมพันธืระหว่างคลาสตั้งแต่สองหรือมากกว่านั้น ที่ถูกระบุโดยการเชื่อมต่อระหว่างออปเจ็คต่างๆ 
ก็จะมี Association Relationship เป็นความสัมพันธ์ที่มีลักษณะระยะเวลานาน และมีความสัมพันธ์แบบโครงสร้าง โดยที่ปลายเส้นความสัมพันธ์ที่ติดกับคลาสจะถูกเรียกว่า Role Name
