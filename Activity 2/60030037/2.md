ในกระบวนการ Data Abstraction ของการเขียนโปรแกรมเชิงวัตถุ (Obejct-Oriented Programming) อันเป็นการกระทำที่จะนำวัตถุที่มีอยู่ในโลกแห่งความเป็นจริงเข้าสู่ระบบคอมพิวเตอร์ โดยมีกระบวนการย่อย ๆ 4 กระบวนการอันได้แก่
1. Classification Abstraction
2. Aggregation Abstraction
3. Generalization Abstraction
4. Association Abstraction

โดยที่การทำ Classification Abstraction จะเป็นกระบวนการในการในแนวคิดของ Object ในโลกแห่งความเป็นจริงที่มีลักษณะเฉพาะเป็นของตนเอง (states and behaviors) และแตกต่างจากสิ่งอื่นเพื่อก่อให้เกิด Class ของ Object นั้น ๆ และเมื่อได้ Class แล้วจะสามารถทำกระบวนการ Instantiation เกิดเป็นวัตถุที่อยู่ในโลกคอมพิวเตอร์ (Instance)

และการทำ Aggregation Abstraction จะเป็นการนำ Class ที่ได้จากการทำ Classification มาประกอบรวมกันเป็นคลาสใหม่ ซึ่งอิงกับโลกแห่งความเป็นจริงที่ว่าวัตถุบางชนิดอาจเกิดขึ้นจากการนำเอาวัตถุหลาย ๆ ชนิดมาประกอบรวมกันเช่น คอนกรีต ได้มาจาก หิน ปูน น้ำ และทราย ซึ่งจัดเป็นกันรวมกันที่ไม่สามารถนำมาแยกออกจากกันในภายหลังได้ และคอมพิวเตอร์ ได้มาจากการนำ เมนบอร์ด cpu powerSupply etc. มาประกอบรวมกัน ซึ่งสามารถแยกออกจากกันได้

หากพิจารณาคอมพิวเตอร์จะเห็นได้ว่าจะต้องมี Cardinality(จำนวนสมาชิกที่ต้องมี) ของ cpu เท่ากับ 1..1 และนอกจากนี้ cpu ยังถือเป็น Required Component (หากไม่มีก็ไม่ถือว่าเป็นคอมพิวเตอร์) และหากพิจารณาอุปกรณ์ต่อพ่วงอื่น ๆ เช่น JoyStick จะถือเป็น Optional Component (จะมีหรือไม่ก็ได้ หากไม่มีก็ยังคงเป็นคอมพิวเตอร์อยู่)
