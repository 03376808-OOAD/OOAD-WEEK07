	ณ ห้องเรียนแห่งหนึ่ง ครูกิ่ง : ถ้าหากถามถึง Classification Abstraction ว่ามันคืออะไร ครูกิ่งจะบอกให้ว่าเป็นกระบวนการที่ให้แนวคิดเพื่อแยกประเภทวัตถุต่างที่อยู่ในขอบเขตที่น่าสนใจ จนได้เป็นคลาส และถือว่าเป็น Abstrsction
ที่สำคัญที่สุด เพราะการแสวงหาในระบบจะเริ่มต้นเกิดขึ้นได้ด้วยการใช้ classification abstraction และข้อมูลพื้นฐานที่จะต้องรู้ก่อนที่จะใช้กระบวนการนี้เนี่ย คือ basic object-Oriented concepts 
object เป็นนสิ่งของที่มีคุณสมบัติ 3 ประการคือ ID (ชื่อ), attribute (ลักษณะประจํา) และ state (สถานะ) ก็คือ สถานะเนี่ยเป็นการบ่งบอกถึงลักษณะประจำของวัตถุ ณ เวลาหนึ่งๆ ส่วน พฤติกรรม (behavior) 
เกิดจากการเรียกวิธีการที่เป็นนิยามของวัตถุในการเปลี่ยนค่าสถานะของลักษณะประจำ 
	และถ้าหากถามถึง Aggregation Abstraction เป็นกระบวนการที่นำเอาคลาสพื้ยฐานมารวมกันหรือประกอบกันเพื่อให้เกิดเป็นคลาสที่ใหญ่ขึ้นหรือซับซ้อนมากขึ้นนั่นเอง
สำหรับ Aggregation Abstraction การประกอบกันของ class หรือความสัมพันธ์เชิง is part of นั้นอาจจะเกิดกรณีหลัก หรือ main class ประกอบไปด้วย class ย่อย ชนิดที่หนึ่ง เพียงชนิดเดียว แต่ประกอบด้วย
คลาสที่สอง จำนวนสี่ชินขึ้นไป และอาจประกอบไปด้วยชนิดที่สาม ไม่จำกัดจำนวน หรืออาจไม่มีเลยก็ได้ ซึ่งสิ่งที่ใช้ในการแสดงความสัมพัธ์ดังกล่าวนี้เรียกว่า cardinality
ในทาง OO เราเรียก class ย่อย ว่า Component นั่นเอง โดยเราเรียกส่วนประกอบที่จำเป็นต้องมีว่า Reqeired Component เช่น รถยนต์ต้องมีเครื่องยนต์ ถ้าไม่มีเครื่องยนต์รถก็วิ่งไม่ได้
เราเรียกส่วนประกอบที่ไม่จำเป็นต้องมีว่า Optional Component เช่น เครื่องปรับอากาศในรถยนต์ไม่จำเป็นต้องมีก็ได้ หมายถถึงไมีมีเครื่องปรับอากาศรถยนต์ก็ยังสามารถวิ่งได้อยู่ พอเข้าใจกันใช่ไหมเด็กๆ
